# Persistent Trust Policy Implementation Plan

> **For the implementing agent:** REQUIRED SUB-SKILL: Use autodev:executing-plans to implement this plan task-by-task.

**Goal:** Add daemon-backed persistent trust grant management to ratchet-cli without duplicating workflow-plugin-agent policy storage.

**Architecture:** Extend the trust RPC group with persisted grant messages and handlers backed by `policy.PermissionStore`. Expose those handlers through client wrappers, `ratchet trust ...` CLI commands, and TUI `/trust` subcommands while keeping existing runtime-only allow/deny behavior unchanged.

**Tech Stack:** Go, gRPC/protobuf generated by `make proto`, Cobra-style manual CLI dispatch in `cmd/ratchet`, Bubbletea command parser tests, workflow-plugin-agent `policy.PermissionStore`.

**Base branch:** `master`
